Integração Make.com (Webhook)
Configure o webhook para receber leads automaticamente dos seus anúncios
Não configurado
Como configurar no Make.com
No Make.com, crie um cenário com o módulo Webhooks → Custom Webhook como gatilho. O webhook receberá um JSON com os dados do lead vindo dos seus anúncios (Meta Ads ou Google Ads). Configure o segundo módulo para enviar uma requisição HTTP POST para este CRM ou use o módulo Firestore diretamente.
JSON esperado pelo webhook (estrutura do lead):
// Estrutura JSON que o Make.com deve enviar ao Firebase
// Collection: "leads" | Método: POST via HTTP Module ou Firestore Module
{
  "name": "{{nome do lead}}",
  "phone": "{{telefone}}",
  "email": "{{email}}",
  "product": "{{produto de interesse}}",
  "source": "{{utm_source}}", // ex: "Instagram", "Google", "Facebook"
  "utm_medium": "{{utm_medium}}", // ex: "cpc", "social"
  "utm_campaign": "{{utm_campaign}}", // nome da campanha
  "utm_content": "{{utm_content}}", // variação do anúncio (A/B)
  "adset": "{{nome do conjunto de anúncios}}",
  "stage": "Novo Lead", // sempre "Novo Lead" para leads de ads
  "city": "{{cidade}}",
  "obs": "Lead via anúncio pago"
}
// O campo "source" (UTM Source) é gravado no Firestore e aparece
// automaticamente na aba Analytics → Taxa de conversão por origem.
